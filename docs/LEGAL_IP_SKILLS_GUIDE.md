# 法律与知识产权 Skills / Plugins 使用手册

日期：2026-06-16

## 一、手册目的

本手册用于说明当前 Codex 环境中与法律、知识产权及相关交付工作有关的本地 skills、相关 plugins 和辅助能力，帮助用户快速判断：

1. 现在有哪些能力；
2. 每个能力适合解决什么问题；
3. 典型输入和输出是什么；
4. 各能力之间如何串联成工作流；
5. 哪些能力已经成熟可用，哪些方向仍存在缺口。

本手册聚焦“如何使用”，不是法律意见本身。

## 二、总体架构

当前能力可分为四层：

1. **基础法律分析层**：负责检索、效力核验、事实梳理、证据结构化、法律推理和风险备忘录。
2. **知识产权专业层**：负责商标、专利、商业秘密、开源软件、合同、平台投诉、警告函等专业任务。
3. **档案与长期管理层**：负责企业知识产权实践档案和 IP 组合生命周期管理。
4. **交付与辅助插件层**：负责正式报告、表格、文档、演示文稿、资料管理和浏览器操作。

建议把它理解成一条工作链：

```text
业务问题
→ 基础法律分析
→ 知识产权专业判断
→ 发函 / 投诉 / 递交 / 管理
→ Word / PDF / 表格 / 演示交付
```

## 四、基础法律分析层

### 1. `cn-legal-source-research`

**用途**：
检索和整理中国大陆现行法律、行政法规、司法解释、部门规章、规范性文件、指导性案例、典型案例及监管材料。

**适用场景**：

- 用户要求查法律依据；
- 用户要求确认“现行”“最新”规则；
- 需要补充权威来源和官方链接；
- 需要形成检索报告。

**典型输入**：

- 问题描述；
- 争议类型；
- 行为发生时间；
- 地域和程序阶段；
- 是否要求最新规则。

**典型输出**：

- 结论摘要；
- 来源表；
- 适用说明；
- 冲突或缺口；
- 待核验事项。

**可以获得什么**：

- 真实、现行、可追溯的法律来源；
- 官方法条链接；
- 可继续进入效力核验和推理的规范清单。

**调用示例**：

```text
使用 $cn-legal-source-research，检索中国大陆商业秘密侵权中举证责任和保密措施认定的现行法律依据，优先官方来源并列出链接。
```

### 2. `cn-legal-norm-validity`

**用途**：
核验法律规范的现行效力、层级、权限、地域范围、时间效力和新旧法衔接。

**适用场景**：

- 判断某条法条是否现行有效；
- 判断行为发生时应适用哪个版本；
- 处理新法、旧法、特别法、一般法冲突；
- 复核他人法律意见中的依据链。

**典型输入**：

- 法律名称和条号；
- 行为发生时间；
- 主体、地域和事项；
- 可能冲突的规范。

**典型输出**：

- 规范效力表；
- 冲突处理结论；
- 时间适用分析；
- 剩余风险。

**可以获得什么**：

- 当前能否引用该规范；
- 应引用哪个版本；
- 是否仅供参考或不应引用。

### 3. `cn-legal-fact-issue-mapping`

**用途**：
把合同、聊天记录、函件、案件叙述等材料转成主体表、时间线、事实状态、法律关系和争议焦点。

**适用场景**：

- 材料杂乱；
- 事实不清；
- 需要把生活语言转成法律分析输入；
- 准备检索、证据分析或诉讼思路。

**典型输入**：

- 合同；
- 邮件和聊天记录；
- 案件经过；
- 时间节点；
- 双方主要主张。

**典型输出**：

- 主体表；
- 时间线；
- 已确认事实 / 单方陈述 / 推断 / 缺失信息；
- 争点清单；
- 待补信息。

**可以获得什么**：

- 后续检索和证据工作真正需要的问题清单；
- 不利事实和矛盾事实不会被提前抹掉。

### 4. `cn-evidence-matrix`

**用途**：
建立主张、构成要件、待证事实、证据、证明力和证据缺口之间的矩阵。

**适用场景**：

- 诉前证据盘点；
- 举证责任分析；
- 维权材料初筛；
- 质证准备；
- 商业秘密、商标、合同等争议的证据断点评估。

**典型输入**：

- 主张与抗辩；
- 程序类型；
- 已有证据材料；
- 待证明的法律要件。

**典型输出**：

- 证据矩阵；
- 证明责任摘要；
- 关键断点；
- 补证优先级；
- 保全建议。

**可以获得什么**：

- 哪些点已经够，哪些点只是“有线索”；
- 哪些证据可以提交、可以采信、可能被质疑。

### 5. `cn-legal-reasoning`

**用途**：
基于已核验的法条、结构化事实和证据状态进行法律推理。

**适用场景**：

- 需要判断请求权是否成立；
- 需要比较双方观点；
- 需要分析抗辩和例外；
- 需要形成可复核结论。

**典型输入**：

- 已核验法条；
- 事实结构；
- 证据状态；
- 需要回答的问题。

**典型输出**：

- 问题；
- 规则；
- 要件分析；
- 反方检验；
- 主结论、备选结论；
- 剩余不确定性。

**可以获得什么**：

- 不是“拍脑袋结论”，而是可追溯到要件和证据的推理链。

### 6. `cn-legal-risk-memo`

**用途**：
把检索、事实、证据和推理整合为正式、简洁、可交付的风险备忘录。

**适用场景**：

- 面向管理层报告；
- 面向客户出内部建议；
- 面向业务团队做决策材料；
- 比较诉讼、谈判、补证、暂缓等路径。

**典型输入**：

- 前述基础 skill 的分析结果；
- 读者对象；
- 决策问题；
- 时间和保密要求。

**典型输出**：

- 执行摘要；
- 风险矩阵；
- 行动选项；
- 时间成本；
- 待核验事项；
- 保留意见。

**可以获得什么**：

- 一份可以进入审批或管理层讨论的正式材料。

## 五、知识产权专业层

### 1. `trademark-clearance`

**用途**：
对中国大陆商标注册可行性进行初筛，识别在先权利冲突、绝对 / 相对驳回风险、恶意注册及驰名商标保护风险。

**适用阶段**：
商标申请前。

**典型输入**：

- 商标标识（文字、图形、组合等）；
- 拟申请商品 / 服务；
- 目标类别；
- 行业、产品和市场范围；
- 是否已有使用。

**典型输出**：

- 注册可行性分析；
- 风险等级；
- 冲突点；
- 类别建议；
- 备选标识或策略建议。

**可以获得什么**：

- 第一轮注册风险评估；
- 是否值得继续做正式数据库检索和申请。

**工作流位置**：

```text
品牌命名
→ $trademark-clearance
→ 正式商标检索 / 申请决策
→ $portfolio 跟踪
```

### 2. `infringement-triage`

**用途**：
对商标权、专利权、著作权和商业秘密侵权纠纷做初步判定。

**适用阶段**：
发现疑似侵权后。

**典型输入**：

- 权利基础文件；
- 被控行为描述；
- 初步证据；
- 被控方信息；
- 维权目标。

**典型输出**：

- 权利基础稳定性评估；
- 侵权行为初步比对；
- 抗辩 / 反诉风险；
- 发函 / 行政 / 诉讼 / 和解建议。

**可以获得什么**：

- 先发函还是先取证；
- 适合行政投诉还是民事诉讼；
- 是否有权利基础不稳或反向风险。

### 3. `cease-desist`

**用途**：
起草知识产权侵权警告函或律师函。

**适用阶段**：
侵权初筛完成后。

**典型输入**：

- 权利基础；
- 侵权事实；
- 希望主张的要求；
- 是否公司名义还是律师函；
- 是否准备后续诉讼或调解。

**典型输出**：

- 警告函草稿；
- 律师函草稿；
- 送达和跟进建议；
- 恶意投诉、商业诋毁、滥用权利风险提示。

**可以获得什么**：

- 一份可进入内部审批的发函文本；
- 对是否发函、何时发函的风险判断。

### 4. `takedown`

**用途**：
起草电商平台、内容平台和网络服务商知识产权侵权投诉通知。

**适用阶段**：
侵权发生在线上平台时。

**典型输入**：

- 权属证明；
- 平台链接；
- 侵权页面截图；
- 比对说明；
- 平台类型。

**典型输出**：

- 平台投诉通知；
- 材料清单；
- 送达与回执保存建议；
- 反通知后的衔接路径。

**可以获得什么**：

- 可直接用于平台投诉的结构化材料；
- 对不同平台规则差异的提示。

### 5. `portfolio`

**用途**：
管理专利、商标、软件著作权、布图设计等 IP 组合全生命周期。

**适用阶段**：
存量资产管理和期限管理。

**典型输入**：

- 资产清单；
- 申请号 / 注册号；
- 类型、日期、状态；
- Excel / CSV / JSON 数据。

**典型输出**：

- 风险分级提醒；
- 年费 / 续展 / 截止日提醒；
- 维持 / 放弃 / 转让建议；
- 组合健康度分析。

**可以获得什么**：

- 哪些商标要续展；
- 哪些专利年费临近；
- 哪些资产值得维持或清理。

### 6. `claim-chart-builder`

**用途**：
构建专利侵权、现有技术抗辩或无效程序的技术特征比对表。

**适用阶段**：
专利侵权诉讼、FTO、无效宣告或被诉答辩。

**典型输入**：

- 权利要求书；
- 被控方案；
- 对比文件 / 现有技术；
- 图纸或说明书。

**典型输出**：

- claim chart；
- 特征比对结论；
- 差异点；
- 现有技术或不侵权抗辩线索。

**可以获得什么**：

- 能用于诉前评估、和代理人进一步工作的结构化比对底稿。

### 7. `fto-triage`

**用途**：
对拟实施的产品或技术方案在中国做自由实施（FTO）初筛。

**适用阶段**：
产品上市、生产、销售或导入前。

**典型输入**：

- 产品技术方案；
- 目标市场；
- 竞争对手或已知相关专利；
- 可公开或内部技术描述。

**典型输出**：

- 高风险专利清单；
- 风险等级；
- 规避设计方向；
- 抗辩线索。

**可以获得什么**：

- 是否存在明显侵权雷区；
- 是否需要升级为正式 FTO 项目。

### 8. `invention-intake`

**用途**：
对专利交底书进行新颖性、创造性、实用性和程序风险初评。

**适用阶段**：
拟申请专利前。

**典型输入**：

- 技术交底书；
- 公开情况；
- 研发背景；
- 目标保护方向。

**典型输出**：

- 三性评级；
- 主题适格性分析；
- 宽限期和保密审查风险；
- 发明 / 实用新型策略建议。

**可以获得什么**：

- 这个技术值不值得走申请；
- 适合申发明、实用新型，还是先补充方案。

### 9. `ip-clause-review`

**用途**：
审查技术合同中的知识产权条款。

**适用阶段**：
合同签约前或争议前梳理。

**典型输入**：

- 合同文本；
- 合作背景；
- 双方角色；
- 是否涉及跨境、委托开发、许可或转让。

**典型输出**：

- 风险点；
- 条款修订建议；
- 权属、许可、保证和违约责任审查意见。

**可以获得什么**：

- 合同中的知识产权陷阱清单；
- 可直接修改的条款方向。

### 10. `oss-review`

**用途**：
审查开源软件许可证合规性。

**适用阶段**：
软件产品开发、交付、融资、客户审计、并购尽调。

**典型输入**：

- 组件清单；
- 许可证类型；
- 集成方式；
- 是否修改源码；
- 是否分发或网络提供。

**典型输出**：

- 合规风险分类；
- 传染性、披露、NOTICE 和专利条款分析；
- 整改建议。

**可以获得什么**：

- 哪些开源组件存在合规压力；
- 是否会影响闭源交付、融资或上市。

### 11. `cnipa-cpc-xml`

**用途**：
审查并转换国家知识产权局发明 / 实用新型递交文件为 CPC-ready XML 包。

**适用阶段**：
专利正式递交前。

**典型输入**：

- 五书 Word 文件；
- 案件目录；
- 图纸和摘要附图；
- 本地模板和成功案例。

**典型输出**：

- 审核意见；
- 修订建议；
- XML 文件夹；
- 最终 ZIP 递交包。

**可以获得什么**：

- 从 Word 材料走到可递交 XML 包的实际交付结果。

## 六、档案与长期管理层

### `cold-start-interview`

**用途**：
为企业建立 `LEGAL_PROFILE.md` 实践档案。

**适用场景**：

- 首次为某企业或团队建立知识产权工作画像；
- 希望后续任务默认继承审批规则、外部服务商、保密分级和风险偏好。

**典型输入**：

- 行业领域；
- IP 组合规模；
- 审批权限矩阵；
- 外部代理和律所；
- 保密分级；
- 开源政策。

**典型输出**：

- `LEGAL_PROFILE.md`

**可以获得什么**：

- 后续各个 skill 可按需读取的企业实践档案；
- 一致性更高的长期工作风格。

## 七、交付与辅助能力

以下能力不是“法律分析 skill”，但与法律 / 知识产权工作强相关：

### 1. `formal-report-docx`

**用途**：
把 Markdown 或结构化内容制作成正式 Word 报告。

**适合**：

- 对客户或管理层交付法律分析报告；
- 输出可编辑的正式 DOCX。

### 2. `formal-report-pdf`

**用途**：
把正式报告制作成经校验的 PDF。

**适合**：

- 对外最终交付；
- 存档版本。

### 3. `markdown-pdf`

**用途**：
快速把 Markdown 转 PDF。

**适合**：

- 内部快交付；
- 轻量打印版。

### 4. `zotero`

**用途**：
管理法规、论文、案例文献和引用。

**适合**：

- 课程、研究、案例笔记；
- 需要引用管理的法学研究工作。

### 5. `Spreadsheets` plugin

**用途**：
处理 IP 资产台账、证据清单、期限表、商标清单和统计表。

**适合**：

- 商标 / 专利组合盘点；
- 年费与续展台账；
- 证据矩阵结构化管理。

### 6. `Documents` plugin

**用途**：
编辑 Word 类文档、批注、生成正式文书。

**适合**：

- 合同审查意见书；
- 律师函、警告函；
- 正式法律备忘录。

### 7. `Google Drive` / `Google Docs` plugins

**用途**：
共享资料、云端协作、Docs 文件交付。

**适合**：

- 团队协作；
- 客户共享版本管理；
- 云端批注。

### 8. `Presentations` plugin

**用途**：
把法律/IP项目分析整理为 PPT。

**适合**：

- 管理层汇报；
- 项目路演；
- 风险通报。

### 9. `Browser` / `Chrome` plugins

**用途**：
浏览、固定网页证据、打开平台规则页面、检查线上页面。

**适合**：

- 平台投诉前查看规则；
- 固定电商页面、商品详情和账号内容；
- 验证本地或远端网页。

## 八、按场景选择 Skill

### 1. 商标注册

```text
$trademark-clearance
```

## 十二、使用建议

1. 简单咨询优先使用一个专业 skill 或一个基础 skill，不必一开始全套调用。
2. 涉及“最新规则、期限、金额、平台规则、现行法”的任务，应同时使用来源检索和效力核验。
3. 涉及维权、发函和诉讼的任务，至少补上事实梳理和证据矩阵。
4. 涉及管理层决策或客户交付时，最后补一层 `cn-legal-risk-memo`。
5. 长期合作的企业，建议先做一次 `cold-start-interview`，建立 `LEGAL_PROFILE.md`。
