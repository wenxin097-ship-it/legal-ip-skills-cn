# 权利要求比对表构建（中国法）

## 目标

为专利侵权诉讼、无效宣告请求或现有技术抗辩，构建结构化、可溯源、符合中国司法实践的技术特征比对表（Claim Chart）。

**适用场景**：
1. **侵权诉讼**：原告/专利权人构建侵权比对表，论证被控侵权产品/方法落入保护范围
2. **无效宣告**：请求人构建无效证据对照表，论证权利要求不具备新颖性/创造性
3. **现有技术抗辩**：被控侵权人构建三方比对表，论证被控方案属于现有技术
4. **不侵权抗辩**：被告构建差异分析表，论证被控方案缺少权利要求技术特征或构成变劣方案

## 可选实践档案

如当前项目存在 `LEGAL_PROFILE.md`，先读取其中与本任务相关的企业背景、审批权限、保密要求和风险偏好，并将其作为补充事实使用。

## 执行前准备

1. 如存在 `LEGAL_PROFILE.md`，仅读取与本次任务相关且已经确认的字段，例如：
   - 企业诉讼/无效策略（维权优先级、外部诉讼律所）
   - 诉讼、无效程序的审批权限与预算阈值
   - 保密规则（诉讼策略属于机密信息，仅限私有化部署）

2. 确认输入材料完整性：
   - 专利文本：权利要求书（全部权利要求）、说明书、附图、审查历史（OA 文件、复审/无效决定）
   - 被控产品/方法：技术规格书、产品手册、现场勘验记录、公证保全材料
   - 无效证据/现有技术：对比文件全文、公开日期证明、翻译件（如为外文）

## 核心法律依据

- 《专利法》第64条：权利要求书是确定保护范围的基础，说明书及附图可用于解释权利要求
- 《最高人民法院关于审理侵犯专利权纠纷案件应用法律若干问题的解释》第1-7条
- 《最高人民法院关于审理侵犯专利权纠纷案件应用法律若干问题的解释（二）》第4-8条
- 《专利审查指南》第二部分第三章（新颖性）、第四章（创造性）——用于无效比对
- 《最高人民法院关于知识产权民事诉讼证据的若干规定》第3-5条（举证责任、证据保全）

## 执行步骤

### Step 1: 模式选择与权利要求解构

#### 模式 A：侵权比对（原告/专利权人视角）

**目标**：证明被控侵权产品/方法的每一个技术特征都能在权利要求中找到对应（相同或等同）。

**权利要求解构原则**：
1. **全部技术特征原则**（司法解释一第7条）：将独立权利要求分解为若干个相对独立的技术特征
2. **前序部分+特征部分**：对发明/实用新型，区分前序部分（已知技术特征）和特征部分（区别技术特征）
3. **功能性特征单独标注**：对以功能或效果限定的技术特征，标注为"FUNC-XX"
4. **用途/环境特征**：对"用于...""在...条件下"等用途或环境限定，标注为"ENV-XX"

**解构示例**（假设权利要求）：
```
一种大气污染物浓度预测方法，其特征在于，包括：
S1: 获取卫星遥感光学厚度数据；
S2: 将所述光学厚度数据与地面监测站数据进行时空融合，得到融合数据集；
S3: 利用深度学习模型对所述融合数据集进行训练，得到浓度预测模型；
S4: 将待预测区域的实时光学厚度数据输入所述浓度预测模型，输出 PM2.5 浓度分布图。
```

**解构结果**：
| 编号 | 技术特征 | 类型 | 说明 |
|:---|:---|:---|:---|
| F1 | 一种大气污染物浓度预测方法 | 主题 | 方法类权利要求 |
| F2 | 获取卫星遥感光学厚度数据 | 步骤 | S1 |
| F3 | 将光学厚度数据与地面监测站数据进行时空融合，得到融合数据集 | 步骤 | S2，注意"时空融合"为核心限定 |
| F4 | 利用深度学习模型对融合数据集进行训练，得到浓度预测模型 | 步骤 | S3，功能性限定"深度学习模型" |
| F5 | 将待预测区域的实时光学厚度数据输入浓度预测模型 | 步骤 | S4 前半 |
| F6 | 输出 PM2.5 浓度分布图 | 步骤/效果 | S4 后半，输出限定 |
```

#### 模式 B：无效证据对照（无效请求人视角）

**目标**：证明权利要求的每一个技术特征被一篇或多篇对比文件公开，或属于公知常识。

**证据组合策略**（审查指南第二部分第四章第3.2.1节）：
- **新颖性**：一篇对比文件（D1）单独公开权利要求的全部技术特征
- **创造性**：D1 + D2 + 公知常识的组合，证明技术方案显而易见
- **注意**：不能使用多篇对比文件组合评价新颖性；创造性组合须存在技术启示

**解构示例**：
| 编号 | 权利要求特征 | D1 公开情况 | D2 公开情况 | 公知常识 | 结论 |
|:---|:---|:---|:---|:---|:---|
| F1 | 方法主题 | D1 公开一种污染物预测方法 | - | - | 公开 |
| F2 | 获取卫星遥感光学厚度数据 | D1 使用卫星数据 | - | - | 公开 |
| F3 | 时空融合 | D1 仅简单拼接，未公开"时空融合" | D2 公开数据融合方法 | - | 有区别 |

#### 模式 C：现有技术抗辩比对（被控侵权人视角）

**目标**：证明被控侵权技术方案的全部技术特征，与一篇现有技术方案中的相应技术特征相同或无实质性差异（《专利法》第67条）。

**比对结构**：三方对照
| 编号 | 权利要求特征 | 被控方案特征 | 现有技术特征 | 比对结论 |
|:---|:---|:---|:---|:---|

### Step 2: 技术特征解释

#### 2.1 字面解释

- 以权利要求书的文字记载为准
- 参考说明书及附图对技术术语的定义
- 参考审查历史（禁止反悔的预判）

#### 2.2 功能性特征解释（关键！）

依据司法解释一第4条、司法解释二第8条：

> 对于权利要求中以功能或者效果表述的技术特征，人民法院应当结合说明书和附图描述的该功能或者效果的具体实施方式及其等同的实施方式，确定该技术特征的内容。

**操作要求**：
1. 识别权利要求中的功能性限定（如"用于...的装置""能够...的机构""深度学习模型"）
2. 在说明书中查找实现该功能的**所有具体实施方式**
3. 将每个实施方式列作对比例
4. 被控方案须与说明书中的任一具体实施方式相同或等同，才落入保护范围

**示例**：
- 权利要求："利用深度学习模型对融合数据集进行训练"
- 说明书实施方式1：CNN（卷积神经网络）
- 说明书实施方式2：LSTM（长短期记忆网络）
- 被控方案使用 Transformer 架构 → 需判断是否属于说明书实施方式的"等同实施方式"

#### 2.3 禁止反悔预判

在构建比对表时，须检查审查历史：
- 申请/实审/复审/无效过程中，申请人是否曾为克服新颖性/创造性缺陷而修改权利要求或陈述意见？
- 若申请人明确将某一技术方案排除在保护范围之外，该方案不得在侵权诉讼中通过等同原则重新纳入（司法解释二第6条）
- 在比对表中，对可能涉及禁止反悔的技术特征，标注"[需核查审查历史]"

### Step 3: 比对分析与结论撰写

#### 3.1 侵权比对结论类型

| 结论 | 定义 | 示例 |
|:---|:---|:---|
| **相同** | 被控方案特征与权利要求特征文字表述一致，或为本领域惯用手段的直接替换 | "获取卫星遥感数据" vs "接收卫星遥感数据" |
| **等同** | 以基本相同的手段，实现基本相同的功能，达到基本相同的效果，且无需创造性劳动即可联想（司法解释一第8条） | 权利要求"CNN"，被控方案"ResNet"（同属深度学习图像处理架构） |
| **不同** | 被控方案缺少该技术特征，或被控方案特征与权利要求特征在手段/功能/效果上存在实质性差异 | 权利要求"时空融合"，被控方案仅简单拼接 |
| **变劣** | 被控方案故意省略个别特征，导致技术效果变差 | 省略"时空融合"步骤，直接单独使用卫星数据 |

**注意**：中国司法实践中，"变劣技术方案"一般不构成侵权（与全面覆盖原则一致）。

#### 3.2 无效比对结论类型

| 结论 | 定义 |
|:---|:---|
| **明确公开** | 对比文件文字明确记载该技术特征，或可直接地、毫无疑义地确定 |
| **隐含公开** | 对比文件虽未明确记载，但属于实施该方案的必然结果或唯一选择 |
| **部分公开** | 对比文件公开了部分特征，剩余部分需其他对比文件或公知常识补充 |
| **未公开** | 对比文件未公开该技术特征，且不属于公知常识 |

### Step 5: 质量检查与风险提示

输出比对表后，必须进行以下检查：

- [ ] **全面覆盖检查**：独立权利要求的每一个技术特征都有对应比对列，无遗漏
- [ ] **功能性特征检查**：所有 FUNC-XX 特征是否已结合说明书具体实施方式解释
- [ ] **禁止反悔检查**：审查历史中是否有排除性陈述未考虑
- [ ] **证据链检查**：被控方案特征是否有充分证据支持（不能仅凭推测）
- [ ] **日期检查**：现有技术/对比文件的公开日是否早于涉案专利申请日
- [ ] **翻译准确性**：外文证据的翻译是否与原文一致（关键术语）

**常见错误警告**：
1. **技术特征划分过粗**：将多个技术特征合并为一个，导致遗漏比对点
2. **功能性特征未解释**：直接按字面比对，未检索说明书实施方式
3. **忽略从属权利要求**：即使独立权利要求不侵权，从属权利要求可能落入保护范围
4. **等同原则滥用**：将差异明显的特征强行认定为等同
5. **证据公开日错误**：将网络公开文章的"发表日"与"网络公开日"混淆（以网络公开日为准）

## 特别注意事项

1. **方法专利的延伸保护**：依据《专利法》第11条，方法专利权的保护延及依照该方法直接获得的产品。在比对时，若被控方案使用了方法专利的步骤，即使产品本身未落入产品专利保护范围，仍可能构成方法专利侵权。

2. **多主体实施方法专利**：通信/软件领域常见"拆分式侵权"（不同主体分别执行方法的不同步骤）。依据司法解释二第21条，明知有关产品系专门用于实施专利的材料、设备、零部件、中间物等，且未经专利权人许可，为生产经营目的将该产品提供给他人实施了侵犯专利权的行为，权利人主张该提供者的行为属于侵权责任法第九条规定的帮助他人实施侵权行为的，人民法院应予支持。

3. **标准必要专利（SEP）比对**：若涉案专利为 SEP，比对时须注意权利要求是否对应标准中的"必选条款"（mandatory）或"可选条款"（optional）。只有对应必选条款的专利才具有不可替代的锁定效应。

4. **图形用户界面（GUI）外观设计比对**：GUI 外观设计保护的是"带有 GUI 的产品的外观设计"，比对时须同时考虑产品类别和 GUI 设计。被控侵权产品须属于相同或相近种类产品，且 GUI 设计相同或近似。
