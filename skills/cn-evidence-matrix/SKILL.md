# 中国诉讼证据矩阵

## 目标

让每项主张都能追溯到待证事实和具体证据，同时暴露证据断点、来源风险和反证。

## 工作流

1. 明确程序类型、主张、抗辩和证明责任分配。
2. 将每项主张拆成构成要件和待证事实。
3. 为每项证据记录持有人、形成时间、取得方式、原件状态和内容摘要。
4. 分析真实性、合法性、关联性、证明目的、证明力及对方可能质证意见。
5. 标注直接证据、间接证据、补强证据、反证和仅供线索材料。
6. 识别缺口并提出合法、必要、可执行的补证路径。

## 约束

- 不把截图、公证、区块链存证或鉴定意见视为当然真实或当然充分。
- 不建议非法取证、越权访问、诱导侵权或侵犯个人信息的取证方式。
- 区分证据存在、证据可提交、证据可采和证明达到标准四个层次。
- 涉及举证责任倒置或转移时引用现行具体条款。

## 输出

输出证据矩阵、证明责任摘要、关键断点、补证优先级和保全建议。

矩阵字段见 [references/evidence-matrix-fields.md](references/evidence-matrix-fields.md)。
